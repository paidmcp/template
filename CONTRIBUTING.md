# Contributing

Thanks for helping improve `paidmcp-template`.

## How to contribute

1. Open an issue first for substantial changes.
2. Keep PRs focused and easy to review.
3. Update docs/examples when behavior changes.
4. Run checks before opening a PR:

```bash
npm run check
npm run format:check
```

## PR checklist

- Clear summary of what changed and why
- Linked issue (if relevant)
- Local checks pass
- No secrets in commits

This repo is solo-maintained, so response times may vary.
